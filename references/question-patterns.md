# Question Patterns

Use one question at a time.

## Code understanding

- What is this file responsible for?
- What depends on this module?
- What happens if this changes?

## Architecture

- Which layer owns this behavior?
- What boundary is this crossing?
- What tradeoff does this design make?

## Components

- What inputs does this component need?
- What outputs or side effects does it produce?
- Where should this logic live instead?

## Workflow

- What is the next smallest step?
- What would you verify before moving on?
- What assumption are we making here?

## Debugging

- Where is the first observable failure?
- What changed most recently?
- What evidence would rule this hypothesis out?
