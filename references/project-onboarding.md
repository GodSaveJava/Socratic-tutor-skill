# Project Onboarding

When a new project starts:

1. Ask whether the user wants `Always On`, `Keyword Trigger`, or `Hybrid`.
2. Ask what success looks like.
3. Identify the main entry point, the primary workflow, and the risky boundaries.
4. Build a simple mental map of the project before discussing implementation details.

## First questions

- What are we building?
- Who will use it?
- What already exists?
- Which part is most confusing right now?

## First map

- Entry points
- Core modules
- Shared utilities
- External dependencies
- Tests and verification path
